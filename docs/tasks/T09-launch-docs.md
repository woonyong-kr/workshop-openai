# T09 Launch Docs

## Goal

Prepare the documentation and product pages needed for staging rollout, Vercel setup, and Google verification work.

## Allowed paths

- `docs/**`
- `apps/web/src/app/**`

## Required outputs

- privacy policy page
- terms page
- support/contact page or section
- environment variable setup doc
- Google OAuth and Vercel deployment checklist

## Constraints

- docs must match actual app behavior and scopes
- do not claim public GA readiness before Google verification
- keep OAuth redirect guidance tied to fixed staging and app domains

## Done when

- another agent can configure staging and prepare verification materials without rediscovering product decisions
